Why Angular + Node.js stack?
  • Angular—Steep learning curve; highly regimented; easy to maintain.
    • TypeScript
      • Popular Microsoft technology (Oct. 2012)
      • Used by 2 of the 3 modern front end technologies (Angular and Vue.js–TypeScript; React.js–JavaScript)
      • Fewer bugs; easy to debug.
    • Backed by Google
    • Highly regimented framework + comprehensive ecosystem (e.g. Material Library) = minimal use of 3rd party libraries; easy to maintain.
    • Widely adopted by large organizations.
    • Steep learning curve compared to React.js.
  • Node.js—JavaScript for back end
    • JavaScript for back end = easy to go full stack. Excellent choice for a small team like ours.
    • Full stack = greatly simplified debugging process.
    • Comprehensive npm packages.
    • Widely adopted and backed by large organizations.
  • sequelize
    • Higher level abstraction = greatly reduced code base.
    • Robust solutions: JavaScript code work on all SQL databases with the correct database engine.
  • MongoDB
    • Greatly reduced code base
    • Vastly simplified data organization when dealing with multi-level hierarchic data structure.
 Learning new technologies by working on projects

When assigning projects, I will take into consideration your current skill set, and assign a project where you can learn new technologies. For example, if you are a back end developer knowlegeable in SQL, I will assign you a project using MongoDB; if you are a front end developer with limited exposure to Reactive Forms, I will assign you a project using Reactive Forms.

If you are interested in learning a particular technology, please let me know so I will accommodate your needs as much as I can.
