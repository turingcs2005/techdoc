SQL

Please use JavaScript (sequelize) in your projects as much as you can, as the syntax is clearer and the code much more concise. Use raw SQL when sequelize library does not provide what you need.

SQL code examples can be found @ git@ssh.dev.azure.com:v3/HanoverSolutionsDevelopers/Solutions%20Developers/SQL_tutorial. Feel free to clone and update the tutorial with your own examples. I have omitted SQL code on complex string manipulation and complex statistics—I believe another language such as Python can get the job done much faster and cleaner. If you are interested in advanced string manipulation and statistics, check out SQL Cookbook (2nd edition, 2020).

The example code are for PostgreSQL. The syntax differs from MS SQL very slightly. Code cover the following topics:

  1. Create sample data for training
  2. Retrieving records
  3. Sorting query results
  4. Working with multiple tables
  5. Inserting, updating and deleting
  6. Metadata queries
  7. Working with strings
  8. Working with numbers
  9. Date arithmetic
  10. Date manipulation
  11. Hierarchic queries

File q0 creates sample data tables for the remaining of the training. Files q1–q10 cover various SQL topics. Most of the code are self-explanatory. You should be able to figure out what the code do by reading the code and reviewing the results. I have also provided explanation in comments.

If you want to use MS SQL, please reach out to administrators at the data engineering team (Eric Naglieri or Daniel Lederman) to set up an MS SQL test database.

To set up PostgreSQL development environment on the Ubuntu Linux virtual machine in Azure, please complete the following:

  1. Ask either me (Rui), Margo or Jeff (or another administrator) to
    • set up your user account
    • create your personal database in PostgreSQL
  2. Install PostgreSQL extension (by Chris Kolkman) in Visual Studio Code.
  3. Create a new connection using your account created in step 1. PostgreSQL is served on default port 5432.
  4. Optionally, if you find the Visual Studio Code extension insufficient, you may use pgAdmin4 installed on the Ubuntu server. pgAdmin4 offers more comprehensive database utilities via GUI (graphical user interface).
  5. When opening a new query file, you will see prompt 'Select Postgres Server' on Visual Studio Code bottom bar. Click to select your database, or you will receive an error message.

Any questions, please reach out to Jeff, James or me.
