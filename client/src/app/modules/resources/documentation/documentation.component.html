Documentation
  • User documentation (how to use the app) should be handled by business partners (subject matter expert).
  • Infrastructure documentation should be handled by an IT professional. Margo can write some generic documentation on Azure.
  • Developers should create technical documentation for a fellow developer of roughly equal skills.
    • Avoid detailed documentation—it becomes stale quickly as code change, and requires a huge effort to keep the documentation up-to-date.
    • Prefered documentation formats:
      • Save key project information such as owner/backup owner, Git repo, deployment history, etc. into a MongoDB database via an internal app.
      • Markdown file for high-level project documentation
      • Material tree component for data structure.
      • .drawio charts for front end artifacts (modules, components, services) to keep track of their main functionalities.
      • .drawio charts for back end tables saved by the app to document schema, primary keys and foreign keys
