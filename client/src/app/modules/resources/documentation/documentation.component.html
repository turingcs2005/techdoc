Documentation
  • User documentation (how to use the app) should be handled by business partners (subject matter expert).
  • Infrastructure documentation should be handled by an IT professional.
  • Developers should create technical documentation for a fellow developer of roughly equal skills.
  • Avoid detailed documentation—it becomes stale quickly as code change, and requires a huge effort to keep up-to-date.
  • Use high-level documentation such as .drawio chart with mouse-over tips.
  • Keep track of main functionalities of each file/module/service/component, etc.
