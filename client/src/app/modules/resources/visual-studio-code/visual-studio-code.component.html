Visual Studio Code
Fira Code Fira Code with ligatures enabled will make your source code more readable by properly displaying multi-character symbols.
  1. Download Fira Code from Google Font
  2. Extract FiraCode-Regular
  3. Control Panel → Font → drop Fira-Code-Regular
  4. Launch Visual Studio Code → Ctrl + Shift + P → Settings (JSON)
  5. Add the following 
        "editor.fontFamily": "'Fira Code', Consolas, 'Courier New', monospace",
    "editor.fontLigatures": true,
Key extensions
  • Remote - SSH
    • Connecting via password
      1. Once installation completes, press F1 or Ctrl + Shift + P to activate quick command
      2. Choose Remote SSH: Connect to Host
      3. When prompted, enter user_name@server_name (for now, YOUR_LAN_ID@lxdvds01.hanover.com)
      4. When prompted, enter your enterprise password.
      5. Navigate to your code directory on Linux server.
    • Connecting via ssh key - no password required
      1. On your laptop, use PowerShell to navigate to C:\Users\YOUR_LAN_ID\.ssh
      2. Generate your private key and public key if you haven't done so
        ssh-keygen -t rsa -b 2048
      3. Copy your public key file to Linux server use either of the two methods below:
        • WinSCP GUI interface
        • PowerShell command
          scp id_rsa.pub YOUR_LAN_ID@lxdvds01.hanover.com:~
      4. Rename your public key file to 'authorized_keys' and change permission to 644
      5. Move file 'authorized_keys' to directory ~/.ssh/
      6. On laptop VSC, create SSH config file
        Host LinuxVM (or whatever server name you prefer)
                HostName lxdvds01.hanover.com
                User YOUR_LAN_ID
                IdentityFile C:\Users\rxw509\.ssh\id_rsa

 Vim extension

Vim is completely optional—it requires months of practice to develop muscle memory. Experienced Vim users can edit source code with improved efficiency and precision; others hate it.

  • Disable Vim's Ctrl + w keyboard shortcut to avoid conflict with VSC's keyboard shortcut: 
    "vim.handleKeys": {
    "<C-w>": false
}
  • Please refer to Vim key binding reference for more information.
 Optional extensions
  • drawio - If you draw visio charts for project documentation. Highly recommended.
  • Docker - If you deploy apps in docker. We are piloting Azure CI/CD pipeline, so deploying from source code will be the future. However, it is still recommended that you install this extension and learn the basics of docker deployment.
  • Emoji Sippets - If you plan to use Emoji in your source code comments.
    After you enter an Emoji, press Tab key so you can enter more Emoji's
  • Python tools - If you are a data scientist and plan to use Python for data manipulation (this scenario is extremely rare in our development work):
    • Python - If you program in Python in VSC.
    • Jupyter - if you use Python notebook for data manipulation.

  • Database extensions for MongoDB PostgreSQL and MS SQL - all these databases come with their own GUI such as MongoDB Compass, pgAdmin4 and MS SQL Server Management Studio, which provide much more sophisticated functionalities. Use VSC extensions for quick access/data review.

    • MS SQL - SQL Server
    • MongoDB - MongoDB for VS Code
 Keyboard shortcuts
  • Ctrl + ~: toggle terminal
  • Ctrl + ,: quick access to VSC settings
  • Ctrl + Shift + p / F1: quick access to command launch
  • Ctrl + Shift + o: show source code hierarchy for quick navigation
  • Ctrl + p: quick search across source code files
  • Alt + tab number: activate said tab, e.g. tab 1, 2, 3
  • Ctrl + w: close active tab (need to disable Ctrl + w in Vim if you have Vim extension installed—see above under 'Vim extension')
  • Ctrl + shift + a: insert inline comment (works under Linux, but not Windows)
  • Ctrl + Shift + [: collapse code block
  • Ctrl + Shift + ]: expand code block
