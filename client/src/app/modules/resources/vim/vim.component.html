Vim
Modes
Mode is displayed at the bottom of the screen. All but search mode is existed (to normal mode) by pressing Esc key; search mode is exited by pressing / Esc.
  • Normal mode: the default and most used mode for moving cursor or selecting test. Activated via Esc key.
  • Insert mode: the second most used mode. Activated via a, A, i, I, o, O keys.
  • Replace mode: replace existing text by directly typing over it. Activated via R key.
  • Visual modes: select text. There're 3 visual modes and they are accessed via
    • v: standard visual model, i.e. selecting text by character
    • V: visual line model, i.e. selecting text by line
    • Ctrl + v: visual block model, i.e. selecting text by block (a rectangular area)
    • gv: start visual mode on the same selected area as the last visual mode
  • Search mode: earch for text/pattern. Accessed via /, ?; exited via / Esc
  • Command mode: perform a wide variety of commands. Accessed via :
 word vs. WORD
By default, a word is a string consists only of:
  • English letter
  • number
  • underscore
In contrast, a WORD is a string consists of anything but whitespace.

For example:
  • 'long-term' = 3 words (long, hyphen, term), but only 1 WORD;
  • 'Tom_Smith' = 1 word and 1 WORD.
Vim key bindings involving word vs. WORD:
  • w - move forward by 1 word
  • W - move forward by 1 WORD
  • e - move to end of word
  • E - move to end of WORD
  • b - move backword by 1 word
  • B - move backward by 1 WORD

Search, replace; copy to another file
Examples:
  • /[pattern]: search for text
  • ?[pattern]: search for text backward
  • n: search again in whatever direction of your last search
  • N: search again in the opposit direction of your last search
  • :%s/[pattern]/[replacement]/gc: replace all occurrences of a pattern with confirmation
    • %: search range, equivalent to 1,$ (whole buffer).
      Can be replaced by 100,120 (search only between lines 100–120)
      When omitted, search is performed on current line only.
    • g: global. search and replace all occurences, not the first occurence
    • c: confirm before replacing
  • select text then "*y to copy
  • navigate to another file and "*p to paste
 Marks
A mark allows you to record your current position so you can return to it later.
  • a–z: file-specific marks. Each file can have its own set of a—z marks.
  • A–Z: global marks. Each mark is unique across all files and is used to identify a position within a particular file.
Examples
  • ma - set local mark a in file
  • mA - set global mark A
  • 'a - jump to first non-whitespace character of line of mark a
  • `a - jump to mark a
  • d'a - delete from current line to line of mark a
  • d`a - delete from cursor to mark a
  • c`a - change from cursor to mark a
  • y`a - yank from cursor to mark a
  • :marks - list all marks
  • :marks aB - list marks a, B
  • :delmarks a - delete mark a
  • :delmarks aB - delete marks a, B
  • :delmarks a-e - delete marks a, b, c, d, e
  • :delmarks! - delete all lower case marks for the current buffer (lowercase marks in other files will not be affected)j
 Command mode
Examples:
  • :q! - close file without saving
  • :w - save file
  • :wq - save and close file
  • :e! - discard all unsaved changes (discard buffer and reopen file on disk)

Key bindings

  • c - change
  • y - yank
  • > - indent
  • < - unindent
Inserting/appending multiple texts
  1. Ctrl + v - enter visual block mode
  2. 3j - go down 3 lines
  3. I; then type ' - prefix ' on multiple lines
  4. Ctrl + o - to execute normal mode command
  5. Ctrl + v - reenter visual block model
  6. $ - move cursor to end of lines
  7. A; then type ' & , - append ' & ,
Incrementing numbers
  1. copy ... 1 ... 5 times
  2. put cursor on 2nd line on number '1'
  3. Ctrl + v, 4j
  4. g, Ctrl + a
